Why this site is boring on purpose

source: site — written for denisbaciu.com
published: 2026-07-06 · status: canonical

I build production systems for a living, so I built this site the way I build those: for the failure modes, not the demo.

The failure mode of a personal archive is time. Platforms rot, get acquired, change terms, or go dark, and fifteen years of writing goes with them. So this site has no framework, no build step, no database, no external fonts, no JavaScript. It is plain HTML and one stylesheet in a git repository. The host is a commodity I can swap in an afternoon. The domain is the only part that matters, and it renews automatically.

Everything I publish anywhere else gets a canonical copy here, with a provenance block like the one above: where it first appeared, when, and a link to the original while the original still exists. Imports are verbatim — an archive that edits its past isn't an archive.

Nothing about this is clever. That's the point. Clever is a dependency.
